Section 1

Project Overview

About TalkPlay

TalkPlay is a speech development web application designed for children aged 3-5 with speech delays. The app helps children practice pronunciation at home through interactive flashcards featuring colorful visuals and engaging audio feedback.

Design Principles

  • Child-First: Large, easy-to-tap targets (minimum 100px for buttons)
  • Joyful: Bright, engaging colors that create positive emotions
  • Clear: High contrast, simple layouts with minimal clutter
  • Encouraging: Positive reinforcement through visual feedback
  • Accessible: Works across devices, easy for small hands

Target Audience

  • Primary: Children aged 3-5 with speech development delays
  • Secondary: Parents and caregivers facilitating speech practice

Key Insight

We chose a vibrant, gradient-rich color palette over traditional educational app aesthetics. Research shows that bold, saturated colors are more engaging for young children and create positive emotional associations with learning activities.

Animations & Interactions

Transition Timing

Speed Duration Usage
Fast 0.15s Button press, active states
Normal 0.3s Hover effects, card movements
Slow 0.5s Progress bar, page transitions

Hover Effects

  • Category Cards: Lift 12px + scale 1.03
  • Icon Buttons: Scale 1.15 + rotate 5deg
  • Back Button: translateX(-4px)

Special Animations

/* Recording Button Pulse */ @keyframes pulse { 0%, 100% { box-shadow: 0 8px 24px rgba(255, 0, 110, 0.3); transform: scale(1); } 50% { box-shadow: 0 12px 36px rgba(255, 0, 110, 0.6); transform: scale(1.05); } } .btn-icon.record { animation: pulse 2s infinite; }

  • Recommended: 100px √ó 100px (our standard)

Andrea's Tasks (Frontend/UI)

  • Create global.css with CSS variables
  • Import Google Font (Fredoka)

  • Configure responsive grid system

Task 2: Home Component (2-3 hours)

  • Build main homepage component structure
  • Create category grid layout
  • Add TalkPlay header with gradient text
  • Implement responsive behavior

Task 3: CategoryCard Component (2-3 hours)

  • Create reusable CategoryCard component
  • Style with category-specific gradients
  • Add icon circles with white background
  • Implement hover effects (lift + scale)
  • Add React Router Link navigation

Task 4: Responsive Testing & Polish (2 hours)

  • Test on mobile (375px width)
  • Test on tablet (768px width)
  • Test on desktop (1200px+ width)
  • Verify touch targets are 100px
  • Check color contrast
  • Verify hover animations work smoothly

Total Estimated Time: 10-13 hours

This meets the 10+ hours per week requirement and aligns with Sprint 1 goals.

Amon's Tasks (APIs/Logic)

Task 1: React Setup & Routing (3-4 hours)

  • Initialize the React project (npx create-react-app)
  • Install and configure react-router-dom
  • Define basic routes in App.jsx (Home, FlashcardPage)
  • Set up project folder structure

Task 2: Data Structure & Mocks (2 hours)

  • Create data/categories.js file
  • Export JSON array with 3 categories (Animals, Foods, Toys)
  • Include category metadata (id, name, color, icon)
  • Add placeholder word data structure

Task 3: LocalStorage Logic Setup (3 hours)

  • Create utils/storage.js helper file
  • Write getProgress() function to read from LocalStorage
  • Write saveProgress(wordId) function to save to LocalStorage
  • Add clearProgress() utility function
  • Test LocalStorage persistence in browser

Task 4: Data Integration & Empty Page (2-3 hours)

  • Connect categories data to Home component
  • Ensure categories render dynamically from data file
  • Create FlashcardPage placeholder component
  • Display "You are in the [Category ID] category" message
  • Verify routing works between Home and FlashcardPage

Total Estimated Time: 10-12 hours

This meets the 10+ hours per week requirement and provides foundation for Sprint 2.
